# Orchestrate

Create and adapt a visible agent topology while preserving one accountable
chair, non-overlapping write ownership and a canonical plan.

## Decomposition test

Delegate only when the proposed slices have:

- distinct questions or artefacts;
- stable enough interfaces;
- independently checkable return contracts;
- no concurrent writes to the same source surface;
- expected value greater than coordination, context and integration cost.

Otherwise work serially or use one specialist.

## Preliminary topology

Before substantial execution, show:

- chair and integration owner;
- each role, provider/family/model intent and effort;
- task, dependencies and completion criterion;
- exact write scope or artefact-only status;
- budget and stop/replan condition;
- review and synthesis plan.

For broad consequential scoping, default to one primary chair and the other
primary as an independent challenge partner. Additional families are optional
and selected for marginal information gain.

## Execution

1. Freeze the approved scope and authority digest.
2. Create a task DAG and reserve non-overlapping workspaces.
3. Issue bounded prompts with evidence and return schemas.
4. Prefer provider-native subagents for local work and Fabric for cross-provider
   authority, work state, messages and receipts.
5. Monitor heartbeat, budget, provider degradation, findings and dependencies.
6. Split, replace, serialise or add agents when evidence warrants it. Record the
   reason and remain inside the approved envelope.
7. Use one integration owner. Workers do not merge, push, publish or deploy
   unless a separate typed effect gate grants that exact action.
8. Synthesize evidence; do not count model votes. Use `investigate` for
   unresolved claims.

## Completion

Return the final topology, task outcomes, substitutions, budgets, evidence,
unresolved risks, review adjudication and canonical hand-off. Scope or authority
expansion returns to the human; routine topology changes do not block progress.

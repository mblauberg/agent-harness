
## 16. Net-current implementation-gate candidate

Spec 05 1.14 remains a freeze candidate. This W017 restatement inventories its
proposed net-current Console implementation gate for verification; it does not
itself authorise material additions beyond v1.0 at `c2fc623` plus directly
traced human clarifications. Subject to that unchanged authority boundary, the
candidate gate preserves:

1. dynamic terminal resizing and stable operator state;
2. bounded evidence UX and exact multi-session selection;
3. answer-bearing Fabric tasks for fresh external review;
4. typed revision-bound review/confirm paths for every shipped Pause/Resume,
   Attention and evidence discussion/request-change action; no required action
   is an implementation placeholder;
5. durable dispatch/read completion within the public protocol deadline;
6. trusted-router admission and daemon-derived review lineage;
7. one complete immutable bundle, linear slot heads, frozen safe results and
   the exact four-slot completion predicate;
8. asynchronous lifecycle-rotation custody, durable target preparation and
   same-agent evidence continuity;
9. authenticated crash-safe Unix-socket supervision, canonical provider action
   pairs and review-diff bytes;
10. capability, annotation, receipt and context-telemetry projections;
11. exact failure receipts, terminal sequence/cut, paged-finding capacity,
    live recovery and resize contracts; and
12. topology/current, route/capability, actual-review-identity and
    context-pressure projections owned by Specs 01-04.

This candidate adds no automatic continuity routing, legacy schema or
compatibility path. The Console owns neither a codec nor policy. Git push,
release, deployment and other separately gated effects remain unauthorised.

# Independent review for high-stakes diagram sets

Use this only when the harness risk tier or user asks for independent review.
Load `orchestrate`; it selects available models and adapters at runtime. Do not
hardcode providers, model IDs or raw provider CLIs in this domain workflow.

## Evidence first

Before dispatching review, run the applicable objective gates:

1. lint the source and record the exact command and result;
2. render from the current source revision;
3. inspect the image for clipping, overlap and ambiguous connections;
4. build the containing document when relevant; and
5. trace actors, relationships and flow outcomes to canonical requirements.

Give reviewers the source, render, applicable project profile and evidence
receipt. Reviewers remain source-read-only and write only their assigned run
artifact.

## Independent lenses

Partition lenses rather than assigning brands:

- UML semantics and requirements traceability;
- visual legibility and source-to-render correspondence;
- domain consistency across registers, descriptions and captions; and
- document integration, accessibility and publication fit where applicable.

The harness chooses scout, workhorse or flagship capacity and the required
family diversity from risk and availability. Failed or skipped lanes stay
explicit. An unavailable distinct family never blocks completion.

## Finding and repair contract

Each finding names the source location or rendered surface, violated contract,
user impact and a verification route. Review conclusions are evidence claims,
not votes. The chair verifies and adjudicates conflicts against the source and
objective checks.

After an authorised repair, rerun lint, render and affected document checks,
then request fresh review of the changed surface. Follow the enclosing delivery
run's repair cap; do not loop until every reviewer prefers the style. Escalate
unresolved semantics or project-profile conflicts to the user.

Completion requires clean objective gates, no unresolved blocking finding and
explicit user acceptance where the enclosing lifecycle requires it.

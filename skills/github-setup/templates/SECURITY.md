# Security policy

Report suspected vulnerabilities privately through:

`<private-reporting-route>`

Maintainers: replace the placeholder before publishing with an enabled GitHub
private-vulnerability-reporting route or another confidential contact method.

Do not put exploit details, credentials, private prompts or personal data in a
public issue.

Include the affected revision, a minimal reproduction, the expected security
boundary, impact and any safe mitigation. Remove real secrets and personal data
from examples.
